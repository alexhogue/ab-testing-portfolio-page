Interface A/B Testing

Goal: to run an A/B test on two different UI designs and analyze the results in order to determine whether a design choice could reasonably impact user interaction

Data Collection

A group of participants was tested as they completed a particular task with two different variations of a webpage interface. The data for both interfaces was collected as the participants completed a particular in order to test the impact of the interface change on the user interaction. The given task was as follows:

Task: Schedule an appointment with Adam Ng, MD at Morristown Medical Center on April 23, 2024

Version A

Interface A has no distinction between the appointment offerings besides the description text itself. Additionally, there is minimal button contrast, which limits the legibility of the buttons used to schedule the appointment.

Version B

Interface B added a background color corresponding to the appointment location. Additionally, the contrast between the button text and background was increased as well as differentiation between the "see appointment" and "schedule appointment" buttons.

Data Analysis

Hypotheses

Misclick Rate

Null hypothesis: A user interacting with interface version A will misclick on the page before completing the task equally on average compared to users of version B.

Prediction: I predict that I will end up rejecting the null hypothesis because I believe the changes made to the interface in version B will alter how users interact with the page, affecting their misclick rate.

Alternative hypothesis: Users interacting with version A will misclick less frequently than users of version B.

Reasoning: I think the changes to the background colors to correspond with the medical center will encourage users to pay more attention to what they click. I also think the button contrast and improved legibility will make the task clearer to users, and reduce their misclick rate. However, it is also possible that the colors confuse the user, resulting in them misclicking.

Time on Page

Null hypothesis: Users of version A will spend the same amount of time on the webpage before completing the task as users of version B.

Prediction: I predict that I will end up rejecting the null hypothesis because I believe the changes made to the interface in version B will alter how quickly users will complete the given task.

Alternative hypothesis: Users of version B will spend less time on the page before completing the task than users of version A.

Reasoning: I think that the changes to the UI of version B, such as the higher contrast buttons and color coded backgrounds, will likely provide visual emphasis to direct users to the target appointment quicker compared to version A.

Mouse Move
Distance

Null hypothesis: Users of version A will move their mouse around the page the same amount as users of version B before completing their task.

Prediction: I predict that I will end up rejecting the null hypothesis because I believe the added button contrast and background color categorization will affect the way in which users navigate down the page.

Alternative hypothesis: Users of version A will move their mouse more on the page before completing the task than users of version B.

Reasoning: I believe the colored backgrounds as well as the boosted contrast and color differentiation between the buttons will provide visual hierarchy to direct the users to the desired appointment more efficiently than version A without these changes.

Statistical Findings

Misclick Rate

For misclick rate analysis, I used a chi-squared test because analyzing the frequency of a particular item in a set of data, such as TRUE and FALSE, is particularly suited for this kind of test; the data collected for misclick rate is binary, with each instance falling into one of two categories: clicked (true) or not clicked (false). Based on the resulting statistics, the difference between versions A and B with respect to this metric is not statistically significant.

  • The degree of freedom of 1 indicates that the chi-squared test involves two categorical variables with two levels each.
  • The small chi-squared value of 0.014 suggests that there is little deviation between the frequencies in the two groups.
  • The high p-value of 0.905 is close to 1, which suggests that any observed differences are likely due to random variation rather than a true distinction between the test groups.

Based on the data from this test, I reject to fail the null hypothesis.

Time on Page

A one-tailed t-test is appropriate when you have a strong directional hypothesis. So, for this metric, I used a one-tailed t-test as I am testing a specific directional hypothesis (whether the mean of one group is significantly greater than or less than that of the other). Based on the one-tailed t-test results, there is insufficient evidence to conclude that the mean of dataset B is significantly less than the mean of dataset A.

  • The degrees of freedom of 48.428 indicates the number of independent observations used to calculate the t-score.
  • The t-score of 0.538 is less than 1, suggesting a relatively small difference between the means of datasets A and B.
  • The p-value of 0.296 represents the probability that group B’s time on the page is less than group A’s time on page. Even though this is less than a 50% chance, in order to have confidence evidence that B < A, this value should be around 0.05. So, there is no strong indication that the mean of A is less than the mean of B.

The high p-value suggests that there is no significant evidence to reject the null hypothesis for this test and suggests that any observed difference in means could likely be due to random variation rather than a true difference between the two datasets.

Mouse Move
Distance

For this metric, I used a one-tailed t-test because the hypothesis predicts a specific directional expectation about the difference between the two groups (that users of version A will move their mouse farther than users of version B). Based on the results of this test, there is evidence to support the hypothesis that the mean of group B is less than the mean of group A. The low p-value suggests that the observed difference in means is unlikely to be due to random variation, and there may be a true difference between the two datasets.

  • The positive t-score of 2.4064 suggests that the mean of group A is higher than the mean of group B. The larger the t-score, the more evidence there is against the null hypothesis.
  • The low p-value of 0.0103 for A > B (less than the commonly used significance level of 0.05) suggests that there is significant evidence to reject the null hypothesis. In other words, there is strong indication that the results of group B are less than the results of group A.

Based on this analysis of the data, I feel confident rejecting the null hypothesis.

Summary Statistics

When running an A/B test, it is important to not only look at the output of the test itself but to also understand the general shape and pattern of the data. One way to do this is through evaluation of summary statistics:

Sample Size:

For the group A tests, 34 data points were collected, and for the group B tests, 30 data points were collected. Increasing the sample spaces of the tests would further improve the accuracy and reliability of the results.

Time on Page
Mean/Median

The median time on page for version A is 8222.5 and the mean is 9729.97. For group B, the median is 8613 and the mean is 8699.79. There is not a mode value for the data as none of the values repeat. These statistics reinforce the failure to reject the null hypothesis. Even though there is a fairly significant difference between the two means, the mean of group A is larger than the median, suggesting a right-skewed distribution. The right skew indicates that there are high values pulling the mean to the right, which can be seen when looking at group A’s individual data points; there is one extreme outlier of 56671. In such cases, the median, being less sensitive to extreme values, gives a more representative measure of central tendency. Thus, the two medians for this metric reflect the similarity in the outcomes of both versions.

Mouse Move
Mean/Median

Version A mouse move distance median is 3853.02 and the mean is 3267.09. For group B, the median mouse move distance is 2450.79 and the mean is 2731.42. For this metric, the median and averages are generally reflective of the analysis of the p-value from the results from the t-test. These values reinforce the rejection of the null hypothesis as the results of group B are consistently smaller than those of group A .

Analyzing these summary statistics not only aids in understanding the trends within each group but also helps to detect unusual patterns that may impact the overall interpretation of the A/B test results. Based on the analyses, I can conclude that the changes made to the interface design do not have a significant impact on user experience; the user in either group made about the same rate of misclicks, and the difference in time spent is not large enough to make a claim about this metric. However, one striking difference between the two groups is difference in the mouse move distance, which was lower for version B. This indicates that the color coded appointments by location or the increased contrast and legibility of the scheduling buttons provided a more direct and clear route to accomplishing the given task. So, while 2 of the 3 metrics did not offer strong enough evidence to reject the null hypothesis, the third metric, mouse move distance, provided interesting data that suggests that there is a quantifiable impact on the user experience due to the change to the interface.

Key Takeaways

A/B testing offers valuable insights into the impact of interface design on user behavior. It emphasizes the need for a holistic evaluation, consideration of multiple metrics, and the importance of specific design elements in influencing user interactions. The reduced mouse move distance for version B users suggests that the changes made to this interface version resulted in a more straightforward user experience. Users were able to navigate and complete the task more efficiently, possibly due to improved visual elements and contrast in the interface. This shows that understanding user behavior patterns, such as mouse move distance, is crucial for optimizing interface design.

The project also underscores the iterative nature of interface design. While some improvement was evident in version B, ongoing testing, user feedback, and further iterations are also necessary to refine and enhance the user experience more significantly.
